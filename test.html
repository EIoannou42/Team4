Heuristics


1. Visibility of system status:


We have a system visibility and its a number based one, it's as your current/total number of pages.

2. Match between system and the real world:


There was not a need for this one since our system is game and not a web that interacts with the players.

3. User Control and Freedom:


We have one and it's only in the landing page, because the game is a fast paced one and you have to answer fast and correctly.

4. Consistency and Standards:


The games consistency and standards are on point, for instance, we place the positive answers on the right and the negative ones on the left.

5. Error Prevention:


There is some sort of error prevention its not throughout the game but in places that is crucial.

6. Recognition rather than recall:


There was not a need for this one since the user has to find the answer, else what is the point of the game then.

7. Flexibility and Efficiency of use:


Same as here there was not a need for this one as well.

8. Aesthetic and minimalist design:


The game is not that big for such functionalities.

9. Help users recognize, diagnose, and recover from errors:


There are not that many possibilities to help the player for the errors since the app is not that complicated.

10. Help and Documentation:


The app provides guidance when it's needed.

User Acceptance


Landing Page:

  • We have rules that when read you will understand how to play the game.
  • The name of the App is displayed on the very top of the page.
  • The two buttons for twitter and facebook are displayed as icons for the user to click on and get redirected.

Application:

  • A clear input bar is displayed for the user to put their username.
  • Instructions in what order to do what in the registration is displayed as well.
  • Pressing the start button will start the quiz.
  • If the user wishes to go back to the home page they can press the button "Back to home page".
  • All buttons for answering have their own name for the user to know what each button does.
  • The number of questions and the current question are displayed on top in order for the user to not lose track of the questions.
  • The score for of the user is displayed for the user to know.
  • Whenever the user gives an answer weather it is correct or not they will be notified of the result (ex. Well done , Wrong).
  • If the user is not in the correct place the app will notify them that they are in the incorrect place.
  • Upon finishing the quiz a loading screen appears so that the user does not think that the app crashed and instead wait for the results.
  • The leaderboard shows all top scores for the user to see.
